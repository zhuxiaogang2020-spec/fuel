### 2. 修复动态 import 问题
**根因**：多处代码使用 `await import('../db/connection.js')` 动态导入，配合 `ts-node-dev --transpile-only` 运行时容易出问题。

**修复文件**：
- `server/src/routes/auth.ts` — 改用 `import pool from '../db/connection'`，query/execute 直接用 pool
- `server/src/routes/records.ts` — 同上，SELECT 用 `pool.query()`（避免 LIMIT/OFFSET 参数化 bug）
- `server/src/middleware/rateLimit.ts` — 改用 `pool.getConnection()`，移除未导入的 `queryOne`

### 3. 修复 `pool.execute()` vs `pool.query()` 
- `pool.execute()` 使用 MySQL prepared statements，LIMIT/OFFSET 参数化时报 "Incorrect arguments to mysqld_stmt_execute"
- 带 LIMIT/OFFSET 的 SELECT 改用 `pool.query()`
- 简单的 INSERT/SELECT 用 `pool.execute()` 仍正常

### 5. stations API 改用数据库优先（前端看不到数据的真正原因）
**问题**：`/api/stations/nearby` 原来只用 MockAdapter（读 fallback_stations.json），完全不查数据库。种子数据虽然在 DB 里但 API 不返回。

**修复**：
- `server/src/routes/stations.ts` — 数据库优先：先 `pool.query('SELECT ... FROM stations')`，计算距离过滤，无数据时回退 MockAdapter
- `server/src/routes/stations.ts` — station-detail 也改成 DB 优先
- 坑：stations 表没有 `rating` 列，SELECT 时要去掉
- 前端 `index.vue` + `auth.ts` 增加详细 console.log 诊断日志
- `auth.ts` 登录失败不再阻塞页面加载

**现在**：`source: "database"`，返回 8 个数据库加油站（按距离过滤后 5-7 个）
